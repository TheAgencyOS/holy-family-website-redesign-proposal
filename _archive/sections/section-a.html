Our default collaboration model for Holy Family. Rick Mitchell's team receives shared Pantheon repository access on day one of kickoff, not at handoff. Code review flows in both directions: we review the university's commits, the university reviews ours, and every pull request has a named reviewer from each side. We hold a weekly 30 minute technical sync with Rick Mitchell, with a rolling agenda document. Documentation is written alongside the code it documents, not assembled at the end, and knowledge transfer is baked into every phase rather than deferred to a Phase 6 training event. By launch, the university's internal team has already maintained parts of the codebase for months.
